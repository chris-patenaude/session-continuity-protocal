# Maintainer Setup & Guide

## Repository Configuration

### Branch Protection
*   **Target Branch:** `main`
*   **Require Pull Requests:** Yes
*   **Require Approvals:** 1 (Recommended)
*   **Status Checks:** Recommended if CI is added later.

### GitHub Features
*   **Discussions:** Enable for Q&A and community support.
*   **Issues:** Use the provided templates.

## Release Process
1.  Update `CHANGELOG.md` with new changes.
2.  Update version number in `README.md` (if applicable).
3.  Create a new GitHub Release tag (e.g., `v1.1.0`).
